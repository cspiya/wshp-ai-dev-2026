# Instruktori dry-run — 2026-07-13 délután (~3 óra)

Cél: a holnapi nap **teljes hallgatói útját** egyszer végigjátszani úgy, ahogy a
résztvevő látja, plusz az instruktori anyagokat időzítve átpróbálni. A végén egy
jegyzőkönyv marad: mi blokkoló (ma este javítandó), mi súrlódás (a teremben szóban
kezelendő), mi kozmetika (ráér).

**Alapszabály:** minden lépésnél mérd az időt és jegyzeteld a súrlódást — pontosan ott
fog elakadni holnap a leglassabb hallgató is, ahol te.

## 0. Előkészítés (10 perc)

1. `git pull` a main-en; ellenőrizd, hogy az utolsó CI-futás zöld (GitHub → Actions).
2. Nyisd meg a publikált site-ot (GitHub Pages) — a nyitóoldal, a menü és a keresés él-e.
3. Készíts egy **tiszta böngészőprofilt** (Chrome/Edge „új profil", bejelentkezések nélkül) —
   ez lesz a szimulált hallgató.
4. Készíts elő egy tartalék e-mail-címet a teszt-fiókokhoz.

## 1. Starter kit hallgatóként (60 perc)

A felkészülés-oldal ( `1.1 Felkészülés a workshopra` ) lépéseit KÖVESD, ne fejből csináld:

1. **Scoop-telepítő** (15 perc): töltsd le a site letöltési linkjéről a
   `scoop-telepito.cmd`-t, futtasd dupla kattintással. Ha a gépeden minden fenn van,
   legalább azt ellenőrizd, hogy a szkript idempotensen lefut és a záró ellenőrzés
   (git/gh/node/claude) zöld. Jegyezd: percek, SmartScreen-figyelmeztetés szövege
   (ezt holnap mondani kell!), bármilyen hibaüzenet.
2. **Fiókok sorrendben** (20 perc, tiszta profilban, tartalék e-maillel):
   GitHub → e-mail megerősítés → Vercel „Continue with GitHub" → Neon GitHub-belépéssel →
   Linear + workspace → v0 Vercel-belépéssel. Jegyezd: hol kér telefonszámot/2FA-t,
   hol van várakozás, melyik UI tér el a leírástól.
3. **Claude Code + MCP** (15 perc): friss mappába klónozd a repót, `claude`, majd `/mcp` —
   mind a négy szerver (Linear, GitHub, Neon, Vercel) OAuth-ja menjen végig. Jegyezd,
   melyik flow hány kattintás, és hol lehet eltéveszteni.
4. **Böngésző-bővítmény** (10 perc): telepítés a tiszta profilba, bejelentkezés, majd
   EGY összekötés végigvitele Claude-dal vezérelve (javasolt: Neon–Vercel integráció) —
   működik-e a kattintás-automatizálás, hol kér jóváhagyást.

**Kimenet:** kipipált felkészülési ellenőrzőlista + időadatok lépésenként.

## 2. Tananyag-út a site-on (45 perc)

1. Napirend-oldal → indulj el a nap útján: `1.8.1` → `1.8.8`, modulonként ~5 perc.
2. Minden modulnál: (a) a Tanulási sorrend lépései végrehajthatóak-e olvasás alapján,
   (b) a Gyakorlat kimenete egyértelmű-e, (c) az Előfeltétel valóban az előző modul
   kimenete-e, (d) lapozó/menü/ábrák rendben.
3. A **2. modul** gyakorlatát (repo felkészítése) CSINÁLD IS VÉGIG egy eldobható
   gyakorlórepóban — ez a nap első és legkritikusabb hands-onja.
4. A **3. modul** spec-csomagját legalább egyszer állítsd elő (az 5 fájl + döntési pont).

**Kimenet:** modulonként max. 1-2 soros jegyzet (OK / súrlódás / blokkoló).

## 3. Referencia-app rendszerpróba (30 perc)

1. `cd reference-app && npm run test` — 12/12 fájl zöld (65 pass, 11 DB-skip normális).
2. `npm run test:e2e` — lokális happy path (in-memory seammel, DB nélkül).
3. `npm run dev` → `/shop`: ár → fake fizetés → regisztráció → megerősítés → lemondás
   kézzel végigkattintva. Auth-panel: mi látszik bejelentkezés nélkül (401-es út)?
4. **Döntés MA:** a Neon Auth élő preview-bejelentkezéshez a `NEON_AUTH_BASE_URL` +
   `NEON_AUTH_COOKIE_SECRET` env a Vercelen még nincs beállítva. Vagy beállítod ma
   (és újrafuttatod a preview e2e-t), vagy holnap a bejelentkezős demó a lokális
   in-memory úton megy, és ezt a 6. modulban őszintén jelzed. Ne maradjon eldöntetlen.

**Kimenet:** shop-út végigjátszva + Neon Auth döntés rögzítve.

## 4. Instruktori forgatókönyv időzített átolvasása (20 perc)

1. Nyisd meg a `nap-forgatokonyv.md`-t, és blokkokra bontva, stopperrel olvasd fel
   magadnak az „Amit elmondasz" vázlatokat — ahol 2 percnél tovább tart egy blokk
   beszédvázlata felmondva, ott húzz.
2. Jelöld meg, hol kell képernyőt váltani (site-oldal / terminál / böngésző), és
   hogy a váltás sorrendje kényelmes-e.
3. Nézd át a „Ha csúszik a nap" szekciót — értesz-e egyet a vágási pontokkal.
