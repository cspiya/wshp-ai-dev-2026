Accountok és MCP-összekapcsolás

A workshopon külső szolgáltatásokat is használunk. Hozd létre (vagy ellenőrizd) az accountjaidat ebben a sorrendben — a későbbiek az előzőkre épülnek:

  1. GitHub — a saját e-mail-címeddel; ez lesz a repóid és a CI otthona.
  2. Vercel — GitHubbal lépj be (deploy és preview).
  3. Neon — GitHubbal lépj be (adatbázis).
  4. Linear — munkaállapot és issue-k.
  5. v0 — opcionális, csak a design-generáló gyakorlathoz.

Az account-létrehozásban a böngészőagent végigvezethet (5. lépés), de bejelentkezést, jelszót és OAuth-jóváhagyást mindig te adsz meg — ez emberi kapu.

MCP-összekapcsolás az agentből. A workshop-repók .mcp.json-ja már tartalmazza a Linear, GitHub, Neon és Vercel MCP-kapcsolatot, titok nélkül. Claude Code-ban a /mcp menüből engedélyezed őket OAuth-tal: a Linear, a Neon és a Vercel így azonnal működik.

GitHub-kivétel. A GitHub MCP OAuth-ja jelenleg nem működik Claude Code-ból (a szerver nem támogat dinamikus kliensregisztrációt). Nem akadály: a GitHub-műveleteket az agent a gh CLI-n keresztül végzi. Ehhez egyszer kell bejelentkezned — kérd meg az agentet, hogy indítsa el a GitHub CLI böngészős bejelentkezését, és a megnyíló oldalon hagyd jóvá. A github MCP-bejegyzés maradhat a konfigurációban; ha az agent kihagyja, az nem hiba.

Lépésről lépésre: három bootstrap-kivétel, aztán agent-run C0

1. Scoop telepítése, ha szükséges

Töltsd le a scoop-telepito.cmd fájlt, és indítsd el dupla kattintással, nem rendszergazdaként. A végén nyiss új terminált. Ez az egyetlen, agent előtti telepítési kivétel.

2. Munkamappa kiválasztása

A fájlkezelőben hozz létre vagy válassz ki egy üres workshop-lab mappát, és nyiss benne terminált. A teljes nap három sibling mappája itt fog élni.

3. Egy agent első indítása

Indítsd el a Claude Code-ot a claude, vagy a Codexet a codex paranccsal. Ez az utolsó ember által begépelt technikai parancs. Egy agent elég. Ajánlott a Claude Code Claude Opus 4.8 modellel; a Pro-kapacitás nem garantált egész napra, ezért a Codex az azonnali tartalék.

4. Doctor, workspace és evidence — az agent feladata

Add az aktív agentnek ezt a természetes nyelvű megbízást:

Prepare my workshop workspace in this selected folder.

Use https://github.com/cspiya/wshp-ai-dev-2026 as the read-only workshop source.
Create sibling workshop-source, participant-repo and workshop-evidence directories.
Read the repository setup guide and agent rules, then run the repository's own doctor
and idempotent bootstrap path. Do not overwrite unknown work, reset files or expose
secrets. Return every command you ran, exit codes, C0 status, remaining risks and
required human approvals. Stop on BLOCKED.

A pontos Git-, PowerShell-, npm- és projektparancsokat az agent olvassa ki a repóból és futtatja. Te azt ellenőrzöd, hogy jó mappában dolgozik, nem ír felül ismeretlen állapotot, és a visszaadott evidence alátámasztja a státuszt.

StátuszJelentésEmberi döntés
PASSA core lokális út valóban futott.Elfogadható C0-nak.
REPLAYAz opcionális lane ellenőrzött manifestből reprodukálható.Csak a bizonyított rész fogadható el.
PENDINGKülső vagy emberi lépés hiányzik.Owner és következő pont szükséges.
BLOCKEDCore előfeltétel hiányzik.Állj meg; ne vidd tovább.

5. Böngészőagent az alapút

Kérd meg ugyanazt az agentet, hogy hivatalos útmutató alapján állítsa be és próbálja ki az elérhető böngésző-integrációt. Claude Code Chrome-ot vagy Edge-et, Codex a Browser vagy Chrome plugint használhat. Bejelentkezést, CAPTCHA-t, OAuth-ot és engedélyt mindig te végzel.

Set up the supported browser-agent path available here. Ask me to perform every login,
CAPTCHA, OAuth or permission choice. Open the participant material and report its
visible title. Mark Connected only after real browser control. After five minutes,
stop and prepare an honest MANUAL fallback instead of claiming automation.

Az ötperces Plan B-ben az ember hajtja végre ugyanazt az utat, az agent pedig segít rögzíteni az URL-t, a látható eredményt és a screenshotot. A jelölés Execution mode: MANUAL; az automatizálási evidence PENDING.

C0 készdefiníció és RUG

  1. Az agent a participant repón futtatta a post-bootstrap doctort.
  2. A saját Git-repó baseline ellenőrzései valóban zöldek.
  3. A workshop-evidence/C0-setup.md felsorolja a futtatott ellenőrzéseket és kilépési kódokat.
  4. Claude Code vagy Codex válaszol a participant repo kontextusában.
  5. A browser mód Connected vagy őszintén jelölt Manual.
  6. Nincs elhallgatott PENDING vagy BLOCKED alapfeltétel.

Vizsgáld meg az agent állításait az evidence alapján. Eltérésnél add vissza pontos korrekcióval és kérj új bizonyítékot. Ez az első AI-val megerősített emberi tanulási ciklus és az első Repeat-Until-Good kapu.
