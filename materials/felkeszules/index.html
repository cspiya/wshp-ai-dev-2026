Felkészülés a workshopra

Cél: a workshop kezdetére legyen működő fejlesztői környezeted, saját gyakorlórepód és egy egyszer már végigfuttatott ellenőrzési útvonalad. A repóba csak nyilvánosan megosztható, kitalált adat kerülhet; minden hálózati függőséghez legyen előre rögzített Plan B.

Mi függ mitől?

Lépésről lépésre: telepítés, fiókok, összekötés

1. Helyi eszközök egyetlen telepítővel (Windows)

  1. Töltsd le a telepítőt: scoop-telepito.cmd.
  2. Indítsd el dupla kattintással — ne rendszergazdaként. A telepítő a Scoop csomagkezelővel felteszi: git, 7zip, gh (GitHub CLI), nodejs-lts, claude-code.
  3. A végén nyiss új terminált, és ellenőrizd: git --version, gh --version, node --version, claude --version — mindegyik fusson le hiba nélkül.

Ha nem Windowst használsz, vagy kézzel telepítenél: Git (git-scm.com), Node.js LTS (nodejs.org), GitHub CLI (cli.github.com), Claude Code (a claude.ai/code útmutatója szerint).

2. Fiókok a saját e-mail-címeddel — ebben a sorrendben

  1. GitHub — github.com: regisztrálj a saját e-mail-címeddel, és erősítsd meg az e-mailt. Minden további fiók erre épül.
  2. Vercel — vercel.com: a „Continue with GitHub” gombbal lépj be — így a GitHub–Vercel kapcsolat azonnal létrejön. A Hobby csomag elég.
  3. Neon — neon.com: itt is a GitHub-belépést válaszd.
  4. Linear — linear.app: regisztrálj, és hozz létre egy saját gyakorló workspace-t.
  5. v0 — v0.app: a Vercel-fiókoddal lépj be.
  6. Claude — claude.ai: aktív Claude Code-hozzáférés kell; ellenőrizd az aktuális csomagfeltételeket.

3. A böngésző-bővítmény és az összekötések

  1. Telepítsd a Claude böngésző-bővítményt Chrome-ba vagy Edge-be (a Chrome Web Store Edge alatt is működik), és jelentkezz be a claude.ai-fiókoddal.
  2. A szolgáltatások összekötését (Neon–Vercel integráció, GitHub-repo importja a Vercelbe, Linear-workspace bekötése) a workshopon a bővítményen keresztül Claude kattintja végig veled — neked csak jóvá kell hagynod a lépéseket. Előre elég annyi, hogy mindenhová be tudj lépni.

4. Claude Code és az MCP-kapcsolatok

  1. Klónozd a gyakorlórepót egy rövid, szóköz nélküli útvonalra, és a mappájában indítsd el a claude parancsot.
  2. A repo előre tartalmazza a négy MCP-kapcsolat (Linear, GitHub, Neon, Vercel) beállítását — telepíteni semmit nem kell.
  3. Írd be: /mcp, és hitelesítsd sorban mind a négy szolgáltatást — mindegyik a böngészőben, OAuth-tal jelentkezik be; kulcsot, tokent sehová nem kell másolnod.

Kész, ha mind a négy MCP-kapcsolat zöld, és az első preview is megnyitható. Titok, jelszó vagy token soha nem kerül a repóba.

Végrehajtható ellenőrzőlista

  1. Jelentkezz be a workshopon használt Git-szolgáltatásba és az AI-fejlesztői eszközödbe.
  2. Ellenőrizd, hogy a Git és a projekt futtatókörnyezete parancssorból elérhető.
  3. Klónozd a saját gyakorlórepódat egy rövid, szóköz nélküli útvonalra.
  4. Futtasd a repo README-jében megadott telepítési és ellenőrzési parancsokat.
  5. Nyisd meg a helyi alkalmazást, majd az első preview környezetet.
  6. Jegyezd fel: melyik parancs futott le, melyik URL nyílt meg, és van-e még blokkoló hiba.

Kész, ha

A repo tiszta állapotú, az alapellenőrzés zöld, a helyi oldal és a preview megnyílik, és nem kell titkot vagy személyes adatot a repóba írnod.
