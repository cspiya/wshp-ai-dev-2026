2. modul · 60 perc

A repo felkészítése AI-agentekkel végzett fejlesztésre

Ebben a modulban a résztvevői repót olyan egyértelmű munkatérré alakítod, ahol egy friss agent chatelőzmény nélkül megérti a célt és a határokat, majd ugyanazokat a helyi ellenőrzéseket tudja futtatni, mint a csapat.

Lényeg: a jó prompt nem pótolja a repo tartós szabályait. A cél, a határok, a készültségi feltételek és a futtatható ellenőrzések együtt teszik biztonságosan művelhetővé a repót.

Előfeltétel

Az 1. modul elfogadott diagnózisa: megnevezett cél, scope, tiltott feltételezések és az első ellenőrzési hiányok. A starter már a saját, írható repódban van; ha nem, kezdd a résztvevői starter határtérképével.

Kimenet

Gyökérszintű agent-utasítás, mérnöki standard, közös Definition of Done és pontos helyi parancsok. Egy friss agent vissza tudja mondani a célt/scope-ot, majd egy sikeres és egy szándékosan hibás ellenőrzést is végigfuttat.

Tanulási sorrend

  1. 0–8 perc Válaszd szét a workshop forrásrepóját, a saját résztvevői repót és az issue-nként elkülönített worktree-ket. Közös ábra: a három munkaterület határa.
  2. 8–20 perc Írd le a repo célját, scope-ját, tiltott műveleteit, nyelvét és azt, mikor kell emberi döntést kérni.
  3. 20–32 perc Készíts mérnöki standardot és Definition of Done listát; minden feltételhez rendelj ellenőrizhető eredményt.
  4. 32–47 perc Adj egy friss agentnek pontos parancsokat a sikeres út, a visszafordítható hibapróba és a helyreállítás végrehajtásához. Tizenöt perc után állítsd meg és rögzítsd, hol tart.
  5. 47–54 perc Kapcsold a munkát issue → branch → worktree rendhez; egy munkadarabnak egy gazdája és egy munkaterülete legyen.
  6. 54–60 perc Helyezd el a CI-t és a preview-t. Ha nincs távoli szolgáltatás, rögzítsd Plan B-ként; ne jelöld PASS-nak.

Gyakorlat: készíts végrehajtható repo-szerződést

  1. A saját repód gyökerében hozd létre vagy pontosítsd az AGENTS.md fájlt: cél, scope, tiltott műveletek, emberi döntési pontok, nyelv, kötelező standardok és parancsok.
  2. Hozd létre a docs/engineering-standard.md fájlt: változtatási méret, tesztelés, review, biztonság és ellenőrzési eredmények.
  3. Írj Definition of Done táblát: minden feltételhez pontos parancs és elvárt eredmény tartozzon.
  4. Nyiss friss agentbeszélgetést chatelőzmény nélkül. Add át neki ezt a teljes próbát, és legfeljebb 15 percet adj a végrehajtásra.
  5. A friss agent futtassa a sikeres utat:
    npm run typecheck
npm run lint
npm run test
npm run build
    Rögzítsd a parancsot, kilépési kódot és rövid eredményt.
  6. A friss agent mondja vissza a repo célját és scope-ját, majd PowerShellben hozza létre a negatív próbát:
    New-Item -ItemType Directory -Force -Path src | Out-Null
Set-Content -LiteralPath src/__workshop-negative__.ts -Value 'const count: number = "not a number"; export { count };'
npm run typecheck
    Elvárt eredmény: FAIL, benne a src/__workshop-negative__.ts pontos helyével.
  7. A friss agent állítsa helyre a repót:
    Remove-Item -LiteralPath src/__workshop-negative__.ts
npm run typecheck
    Elvárt eredmény: a fájl eltűnik és ugyanaz a typecheck ismét PASS.

Kész, ha: a friss agent a repóból helyesen visszamondja a célt és a határokat; a négy normál parancs sikeres; a szándékos típushiba megállítja a kaput; törlés után ugyanaz a kapu ismét sikeres.

Plan B és megállási szabály

Ha CI vagy preview nem érhető el, rajzold fel a kitalált helyi topológiát és futtasd a helyi ellenőrzéseket. A távoli szolgáltatás eredménye legyen NOT RUN — service unavailable vagy BLOCKED, soha ne PASS. Ha a sikeres helyi út sem fut, a modul nincs kész.

Átadás a 3. modulnak

Az elfogadott repo-szabályok, Definition of Done és pontos parancsok korlátozzák a 3. modul specifikációját. Segédletek: repo-szabályok és automatizált kapuk. Előkészítés: résztvevői felkészülés.
