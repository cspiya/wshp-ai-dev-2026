not what is asserted. Three navigation lines were added; **every original
assertion and expected value (incl. `10287 HUF`) is unchanged**:

1. `await page.getByRole("button", { name: "Select & continue" }).first().click();`

   — acknowledges the step-02 account step (runs signed out locally, after
   the unchanged preview sign-up block when targeting a deployment).
3. `await page.getByLabel("List price (minor units)").fill("10000");`
   — the quote's list price is now prefilled from the SELECTED workshop, so

   exact `10287 HUF` expectation deterministic regardless of catalog order.

## 14. Honest limitations and residual risks

- The step-02 "Continue to price & payment" acknowledgement and the selected
  workshop id are UI navigation state, not backend state — deliberately, per
  the brief ("a visual state may be documented; the integrated UI must be
  driven by the existing real query/mutation state"). The account chip
  honestly reads "Signed out · local demo" when no session exists.
- The mock's "Seats left x/y" catalog readout was NOT ported: no registration
  count exists in the API; the card shows capacity instead. Same for the
  sold-out card state.
- The certificate's "cancel window: closes 48h before start" is static copy
  of the module default; the window is deployment-configurable at the
  composition root and the client cannot read the effective value.
- The quote-error evidence shows the raw Zod message JSON from tRPC —
  pre-existing behavior (the UI renders `quote.error.message` as-is),
  kept rather than reformatted in the presentation layer.
- Evidence screenshots were taken on the dev server, so the Next.js dev-tools
  badge (dark "N") is visible in some corners; visuals are otherwise
  identical to the production build the gates exercised.
- Preview-targeted e2e (`PLAYWRIGHT_BASE_URL`) was reasoned about (section
  13) but not executed in this session — no preview deployment was in scope.
- Dark-mode tokens remain defined but unreachable (no toggle); the accepted
  design is a deliberate single-appearance system.
