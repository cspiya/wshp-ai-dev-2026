# AI-generated test review

- [ ] The test can fail for the intended defect (mutation/negative sanity check where practical).
- [ ] Assertions verify behavior, not only that execution completed.
- [ ] Important denial, invalid, empty, duplicate, and dependency-failure paths are covered.
- [ ] Test data is synthetic and deterministic.
- [ ] Mocks do not hide the behavior under test.
- [ ] Fake and real adapters share contract tests where drift matters.
- [ ] Time, locale, ordering, and concurrency assumptions are explicit.
- [ ] The test ran in the claimed environment; skips and filters are visible.
- [ ] Browser/e2e checks exercise the real user boundary where required.
