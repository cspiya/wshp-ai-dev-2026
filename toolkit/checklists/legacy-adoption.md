# Legacy and adoption minimum

- [ ] Establish characterization tests before changing poorly understood behavior.
- [ ] Identify one safe seam and one small vertical slice; avoid rewrite-by-default.
- [ ] Protect database/schema changes with backup, migration, compatibility, and rollback plans.
- [ ] Preserve authorization, audit, and operational constraints.
- [ ] Run old and new paths against representative synthetic data where possible.
- [ ] Introduce the workflow with one team, one repository, and measurable gates.
- [ ] Define ownership for specs, standards, review, CI, and exceptions.
- [ ] Track lead time, escaped defects, review findings, and agent cost without turning metrics into quotas.
- [ ] Expand only after the pilot has repeatable evidence and a documented fallback.
