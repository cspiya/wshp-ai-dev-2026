# Context budget

- [ ] Give the agent the task, approved spec, local rules, relevant files, and canonical standards—not the whole repository.
- [ ] Keep one role and one bounded outcome per context.
- [ ] Start reviewers in fresh contexts; do not include maker reasoning or verdicts.
- [ ] Prefer links/paths to one canonical source over copied instruction blocks.
- [ ] Use deterministic scripts for repeatable checks instead of explaining their logic in every prompt.
- [ ] Summarize decisions and evidence before context rollover.
- [ ] Start a new context when the task changes, stale assumptions accumulate, or independent judgment is required.
