# Spec gate

- [ ] The problem and observable outcome are explicit.
- [ ] In-scope and out-of-scope behavior are separated.
- [ ] Actors, authorization, data, and system boundaries are known.
- [ ] Acceptance criteria are testable and use Given–When–Then where useful.
- [ ] Important failures, edge cases, compatibility, locale/time, and concurrency are addressed.
- [ ] Evidence required for done is named.
- [ ] Open product decisions have an owner; implementation does not guess them.
- [ ] A human owner approved the contract.
