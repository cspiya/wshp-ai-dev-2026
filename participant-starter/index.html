Biztonságos kiindulópont

Résztvevői starter

Egy szándékosan kicsi Next.js projektet kapsz. Az alkalmazás elindul és vannak ellenőrzési parancsai, de a csapatod működési szabályait, specifikációját és későbbi szolgáltatásait még neked kell hozzáadnod.

Lényeg: a starter technikai alap, nem kész fejlesztési rendszer és nem kész termék.

Melyik repóban dolgozz?

Workshop forrásrepo · csak olvasd és másold

Itt találod a tananyagot és a starter eredeti példányát. Ebben a repóban ne végezd el a gyakorlatot, és ne írd felül a közös workshop-forrást.

Saját résztvevői repo · itt dolgozz

Másold ide a startert. Itt alakítsd ki a szabályokat és a specifikációt, itt módosítsd a kódot, és itt rögzítsd az ellenőrzési eredményeket.

Határ: a workshop forrásrepo a kiindulópont; a saját repód az írható munkatér. A preview, az API és az adatbázis csak a megfelelő későbbi modulban kapcsolódik hozzá.

Ami már készen áll

  • Next.js App Router, TypeScript és Tailwind alap.
  • Egy kicsi, futtatható példaoldal és mintateszt.
  • typecheck, lint, test és build parancs.
  • Kiinduló repo- és dizájnszabályok, amelyeket a saját projektedhez igazítasz.

Ami szándékosan nincs benne

  • Nincs kész üzleti funkció vagy előre eldöntött specifikáció.
  • Nincs adatbázis, API-kapcsolat vagy valódi környezeti titok.
  • Nincs helyetted elfogadott repo-szabályrendszer.
  • Nincs automatikus bizonyíték arra, hogy a csapatod munkamódja működik.

Első biztonságos lépések

Előfeltétel: a starter már a saját, írható repódban van, és legalább Node.js 22.13 fut a gépeden. A workshop CI-folyamata is a 22-es főverziót használja; régebbi 22.x verziónál egyes fejlesztői függőségek figyelmeztetést adnak.

  1. Ellenőrizd a környezetet.
    node --version
npm --version

    Elvárt eredmény: mindkét parancs verziószámot ír ki.

  2. Telepítsd a zárolt függőségeket.
    npm ci

    Elvárt eredmény: a parancs hibakód nélkül befejeződik. Ne másolj titkot az .env.example fájlba.

  3. Futtasd a teljes indulás előtti ellenőrzést (preflight).
    npm run typecheck
npm run lint
npm run test
npm run build

    Elvárt eredmény: mind a négy parancs sikeres. A teszt jelenleg egy szándékosan kicsi mintát ellenőriz.

  4. Olvasd el, majd igazítsd a saját repohoz az AGENTS.md és DESIGN-GUIDELINE.md fájlokat. Ne tekintsd késznek az általános kiinduló szöveget.
  5. Állj meg az üzleti funkció fejlesztése előtt. A következő modulban rögzítsd a repo célját, határait és kötelező ellenőrzési pontjait.

Sikerfeltétel: egy friss agent chatelőzmény nélkül meg tudja mondani, hol dolgozhat, mely parancsokat kell futtatnia, és mely döntéseket nem hozhatja meg helyetted.

Ha elakadsz: hálózati vagy telepítési hiba esetén használd a tréner által megadott commitazonosítóhoz (SHA) tartozó, kitalált adatokkal készült ellenőrzési eredményt. Jelöld REPLAY-ként, vagyis visszajátszott eredményként; ne állítsd, hogy helyben futtattad.
